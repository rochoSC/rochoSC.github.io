CSCE 666 - Pattern Analysis

We made a prediction model based on accelerometer information with aims to predict hearth rate. The goal was to reduce the "PPG/ECG" energy consumption from wearable devices in order to make the HR data collection more reliable for large periods of time. CSCE 670 - Information Retrieval

We implemented a "Deep" neural network as a recommender system. Our work was an extension of an existent publication offering recommendation using DNN (He, Xiangnan, et al. "Neural collaborative filtering." Proceedings of the 26th international world wide web conference. 2017). We extended their work by incorporating richer information when available, such as textual reviews and movie metadata to further increase accuracy rates. We performed two additional experiments in addition to replicating the original paper. The first experiment utilized explicit ratings from data, such as the 1-5 star ratings commonly found in reviews. The second used Paragraph2Vec to create paragraph embeddings which will capture the textual content of any reviews. We used Python and Keras library for the DNN development.


CSCE 634 - Intelligent User Interfaces

We created an Android app that classifies user emails as either stressful or not based on sentimental analysis of the email's content. Then, according to their current stress levels inferred from their current respiration rate provided by the Bioharness, we offered the user visual aid to let them know whether an email is likely to stress him out more if he reads it in that moment. In other words, we try to help the user to not get more stressed by letting him know that a certain email may represent stressful information. We used Android/Java for the client side and Flask framework for the Python back-end.
